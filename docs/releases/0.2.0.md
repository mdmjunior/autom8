# AutoM8 0.2.0

Release estável da versão 0.2.0 do AutoM8.

Esta versão inaugura o módulo Apps com catálogo local, categorias, perfis e fluxo de validação para Ubuntu Desktop.

## Destaques

- Catálogo de apps separado por categorias.
- Catálogo consolidado em `suite/catalog/apps.json`.
- Validação forte do catálogo de apps.
- Instalação e remoção individual de apps.
- Instalação e remoção múltipla.
- Instalação e remoção por categoria.
- Apps com status `advanced` e `planned` são bloqueados em ações automáticas.
- Perfis baseados no catálogo.
- Perfis iniciais:
  - `dev-essential`
  - `server-basic`
  - `desktop-basic`
  - `creator`
  - `container-basic`
  - `gamer-basic`
- Smoke test para Ubuntu Desktop.
- Validação pós-instalação de RC.
- Build do site protegido contra instalador divergente.
- Installer canônico em `installer/install.sh`.
- `site/public/install.sh` sincronizado e validado a partir do canônico.
- Empacotamento com validação de catálogo antes da geração do tar.gz.

## Comandos principais

Apps:

    autom8 apps categories
    autom8 apps list
    autom8 apps list --category sistema
    autom8 apps show git
    autom8 apps install git
    autom8 apps install-many git htop jq
    autom8 apps install-category sistema
    autom8 apps remove git
    autom8 apps remove-many git htop
    autom8 apps remove-category sistema

Perfis:

    autom8 profiles list
    autom8 profiles show dev-essential
    autom8 profiles install dev-essential
    autom8 profiles remove dev-essential

Dry-run:

    autom8 --dry-run apps install-category sistema
    autom8 --dry-run profiles install dev-essential

## Instalação

Após publicada a release estável:

    curl -fsSL https://autom8.oslabs.com.br/install.sh | bash

## Requisitos

- Linux com apt, dnf, zypper ou pacman.
- Usuário comum com sudo.
- Internet disponível.
- Ubuntu Desktop validado como ambiente principal de teste para esta release.

## Observações

- Apps `advanced` aparecem no catálogo, mas não são instalados automaticamente nesta versão.
- A instalação e remoção real exigem confirmação explícita.
- Perfis reutilizam os fluxos de instalação/remoção múltipla de apps.
- Use `--dry-run` antes de ações reais em ambientes importantes.
