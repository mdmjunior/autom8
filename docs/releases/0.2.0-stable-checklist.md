# Checklist de promoção — AutoM8 0.2.0 estável

Este checklist promove `v0.2.0-rc2` para `v0.2.0` estável apenas se o RC foi aprovado em VM.

## Pré-condição obrigatória

O arquivo de resultado do teste deve existir e indicar aprovação:

    docs/releases/results/0.2.0-rc2-ubuntu-desktop.md

Critérios mínimos:

- smoke passou;
- instalação via pacote RC passou;
- `autom8 --version` mostrou 0.2.0-rc2;
- `autom8 doctor` executou;
- apps funcionaram;
- profiles funcionaram;
- dry-runs não alteraram sistema;
- app advanced foi bloqueado;
- não há bug crítico aberto.

## Preparar branch

    git checkout feature/apps-v0.2
    git pull --ff-only origin feature/apps-v0.2
    git status

Working tree deve estar limpo.

## Ajustar versão

    printf '0.2.0\n' > suite/VERSION

## Sincronizar e validar

    ./scripts/build-apps-catalog.sh
    ./scripts/validate-apps-catalog.sh
    ./scripts/validate-profiles-catalog.sh
    ./scripts/verify-installer.sh
    ./scripts/smoke-ubuntu-desktop.sh
    ./scripts/build-site.sh
    ./scripts/package.sh

## Commit da versão estável

    git add suite/VERSION docs/releases/0.2.0.md docs/releases/0.2.0-stable-checklist.md
    git commit -m "Prepare v0.2.0 stable release"
    git push

## Merge para main

    git checkout main
    git pull --ff-only origin main
    git merge --no-ff feature/apps-v0.2 -m "Merge AutoM8 v0.2.0"
    git push origin main

## Validação Fedora obrigatória antes da stable

Antes de publicar `v0.2.0`, validar Fedora Workstation:

    ./scripts/smoke-fedora-workstation.sh

Em VM Fedora descartável, testar também o RC2 via instalador:

    curl -fsSL https://autom8.oslabs.com.br/install.sh -o /tmp/autom8-install.sh
    AUTOM8_PACKAGE_URL=https://github.com/mdmjunior/autom8/releases/download/v0.2.0-rc2/autom8-0.2.0-rc2.tar.gz bash /tmp/autom8-install.sh

Critérios mínimos Fedora:

- CLI instala em `/opt/autom8/bin/autom8`;
- `autom8 --version` mostra `0.2.0-rc2`;
- apps básicos em dry-run funcionam;
- app advanced é bloqueado;
- perfis listam;
- eventuais falhas exploratórias são registradas.


## Publicar release estável

Na main:

    ./scripts/release-stable.sh

Assets esperados:

    autom8-0.2.0.tar.gz
    autom8-latest.tar.gz

## Validar release

    gh release view v0.2.0 --repo mdmjunior/autom8 --json tagName,name,isPrerelease,url
    gh release view v0.2.0 --repo mdmjunior/autom8 --json assets --jq '.assets[].name'

## Validar instalador público

Em VM Ubuntu Desktop descartável:

    curl -fsSL https://autom8.oslabs.com.br/install.sh | bash

Depois:

    export PATH="/opt/autom8/bin:$PATH"

    autom8 --version
    autom8 doctor
    autom8 apps categories
    autom8 profiles list
    autom8 --dry-run apps install-category sistema
    autom8 --dry-run profiles install dev-essential

## Pós-release

- confirmar site publicado;
- confirmar download do latest;
- criar tag/registro interno;
- abrir planejamento v0.2.1 ou v0.3.0;
- registrar bugs encontrados após release.
