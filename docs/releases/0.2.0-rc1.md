# AutoM8 0.2.0-rc1

Release candidate da versão 0.2.0 do AutoM8.

Esta versão é uma prerelease para testes em VM, antes do merge final na main e antes da publicação estável.

## Destaques

- Catálogo de apps separado por categorias.
- Catálogo consolidado em `suite/catalog/apps.json`.
- Validação forte do catálogo de apps.
- Instalação e remoção individual de apps.
- Instalação e remoção múltipla.
- Instalação e remoção por categoria.
- Bloqueio automático de apps `advanced` e `planned`.
- Perfis baseados no catálogo.
- Perfis iniciais:
  - `dev-essential`
  - `server-basic`
  - `desktop-basic`
  - `creator`
  - `container-basic`
  - `gamer-basic`
- Smoke test para Ubuntu Desktop.
- Build do site blindado para não sobrescrever instalador com versão quebrada.
- Installer canônico em `installer/install.sh`.
- `site/public/install.sh` sincronizado e validado a partir do canônico.
- Package script validando catálogos antes de empacotar.

## Teste recomendado

Em uma VM Ubuntu Desktop descartável:

    sudo apt update
    sudo apt install -y git curl jq

    git clone -b feature/apps-v0.2 https://github.com/mdmjunior/autom8.git
    cd autom8

    ./scripts/smoke-ubuntu-desktop.sh

## Teste do instalador com pacote RC

O instalador público ainda aponta para a release estável latest. Para testar este RC, use override de pacote:

    curl -fsSL https://autom8.oslabs.com.br/install.sh -o /tmp/autom8-install.sh
    AUTOM8_PACKAGE_URL=https://github.com/mdmjunior/autom8/releases/download/v0.2.0-rc1/autom8-0.2.0-rc1.tar.gz bash /tmp/autom8-install.sh

## Avisos

- Esta é uma prerelease.
- Use apenas em VM descartável ou ambiente de teste.
- Ainda não é a release estável v0.2.0.
- Apps `advanced` aparecem no catálogo, mas não são instalados automaticamente.
- O fluxo público `curl ... install.sh | bash` só deve ser usado para validar v0.2 após a release estável.
