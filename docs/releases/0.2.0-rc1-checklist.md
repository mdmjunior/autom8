# Checklist AutoM8 0.2.0-rc1

## Antes de publicar o RC

    git checkout feature/apps-v0.2
    git pull --ff-only origin feature/apps-v0.2
    git status

O working tree deve estar limpo.

## Validações locais

    bash -n scripts/release-rc.sh
    bash -n scripts/smoke-ubuntu-desktop.sh
    bash -n scripts/package.sh
    bash -n scripts/build-site.sh

    ./scripts/smoke-ubuntu-desktop.sh
    ./scripts/build-site.sh
    ./scripts/package.sh

## Publicar prerelease

    ./scripts/release-rc.sh

## Validar assets

    gh release view v0.2.0-rc1 --repo mdmjunior/autom8 --json tagName,name,isPrerelease,url
    gh release view v0.2.0-rc1 --repo mdmjunior/autom8 --json assets --jq '.assets[].name'

Assets esperados:

    autom8-0.2.0-rc1.tar.gz
    autom8-rc-latest.tar.gz

## Testar em VM Ubuntu Desktop por clone

    sudo apt update
    sudo apt install -y git curl jq

    git clone -b feature/apps-v0.2 https://github.com/mdmjunior/autom8.git
    cd autom8

    ./scripts/smoke-ubuntu-desktop.sh

## Testar instalador com pacote RC

    curl -fsSL https://autom8.oslabs.com.br/install.sh -o /tmp/autom8-install.sh
    AUTOM8_PACKAGE_URL=https://github.com/mdmjunior/autom8/releases/download/v0.2.0-rc1/autom8-0.2.0-rc1.tar.gz bash /tmp/autom8-install.sh

## Validar instalação RC

    export PATH="/opt/autom8/bin:$PATH"

    autom8 --version
    autom8 doctor
    autom8 apps categories
    autom8 apps list --category sistema
    autom8 profiles list
    autom8 --dry-run apps install-category sistema
    autom8 --dry-run profiles install dev-essential

## Critérios para aprovar o RC

- Release marcada como prerelease.
- `autom8 --version` mostra `0.2.0-rc1`.
- Doctor executa sem quebrar a CLI.
- Apps lista categorias.
- Apps mostra detalhes.
- Apps bloqueia `advanced`.
- Perfis listam e detalham.
- Dry-runs não alteram sistema.
- Instalação real controlada funciona em VM descartável.

## Verificação pós-instalação

Depois de instalar o RC em `/opt/autom8`, rode:

    AUTOM8_BIN=/opt/autom8/bin/autom8 AUTOM8_EXPECTED_VERSION=0.2.0-rc1 ./scripts/verify-installed-rc.sh

## Registro do resultado

Copie o template:

    cp docs/releases/results/0.2.0-rc1-ubuntu-desktop-template.md docs/releases/results/0.2.0-rc1-ubuntu-desktop.md

Preencha:

    docs/releases/results/0.2.0-rc1-ubuntu-desktop.md

## Decisão pós-RC

Se houver bug crítico:

    Corrigir na feature/apps-v0.2
    Atualizar suite/VERSION para 0.2.0-rc2
    Gerar nova prerelease

Se aprovado:

    Atualizar suite/VERSION para 0.2.0
    Sincronizar docs
    Merge em main
    Publicar release estável
